Project 02

설계 가능한 데이터 조직 빌딩과 AI 전환

이 프로젝트는 데이터 팀을 단순 적재 반복 조직에서 기술 기반의 설계 전문 조직으로 전환하는 과정이었습니다. 저는 팀의 역할과 KPI를 다시 정의하고, 운영 현장에서 발생하는 요구가 데이터 제품과 AI 기반 플랫폼 과제로 이어지도록 구조를 설계했습니다.

Problem

수작업 적재 중심 구조

데이터 팀이 반복 적재를 중심으로 운영되면서 데이터의 목적성이 약했고, 유관 부서의 요구를 선제적으로 설계 수준에서 대응하기 어려운 한계가 있었습니다.

Role

팀 체계와 운영 기준 재설계

약 20명 규모의 데이터 조직을 기준으로 역할 체계, 운영 기준, 협업 방식, 문제 해결 흐름을 다시 정리하는 역할을 맡았습니다.

Goal

설계와 자산화가 가능한 조직으로 전환

단순 요청 처리 조직이 아니라, 데이터가 자산화되고 AI와 플랫폼 전략으로 이어질 수 있는 구조를 만드는 것이 목표였습니다.

핵심 실행 내용

조직 전환은 이름을 바꾸는 일로 끝나지 않았습니다. 역할의 단위를 바꾸고, 성과를 측정하는 방식을 세우고, 다른 부서와 맞닿는 인터페이스까지 함께 설계해야 실제 변화가 일어났습니다.

Action 01

4개 역할 축으로 팀 구조 재정의

  • Project: 로드맵과 우선순위 설계
  • Quality: 검수와 표준 기준 정비
  • Contents: 도메인 전문성 강화
  • Product: 시스템과 자동화 방향 정의
Action 02

KPI 기반 운영 시작

  • 공정 리드타임
  • 오류율
  • 자동화율
  • 재작업률
Action 03

다학제적 조직 운영

엔지니어, PM, QA, 운영 매니저 등 다양한 직군이 하나의 문제를 공통 기준으로 풀 수 있도록 협업 방식과 역할 경계를 정리했습니다.

Action 04

AI 전환의 기반 마련

단순 적재에서 끝나는 흐름을 줄이고, 데이터가 ML, 콘텐츠, 서비스 조직의 요구와 연결될 수 있도록 구조적 기반을 만들었습니다.

성과와 의미

Result

데이터 자산화의 기반 마련

반복 작업 중심의 운영에서 벗어나 데이터의 목적과 활용 방향을 설계할 수 있는 상태로 전환했습니다.

Result

선제 대응 가능한 구조 확보

ML, 콘텐츠, 서비스 조직의 데이터 수요를 사후 대응이 아니라 설계 단계에서부터 함께 다룰 수 있게 되었습니다.

Meaning

AI 데이터 플랫폼 전환의 출발점

이후의 AI-Data Team 전환과 플랫폼 방향 수립은 이 조직 구조 재정의 작업이 있었기에 가능했습니다.
